# 架构设计技能

## 何时使用

- 设计系统架构
- 选择技术栈
- 评估技术风险
- 制定技术决策

## 项目标准

> 📖 参考 → `.claude/project_standards.md`

## 工作流程

1. **分析需求** - 理解 PRD，识别挑战
2. **架构设计** - 选择模式，定义组件
3. **技术选型** - 参考 project_standards.md
4. **API 设计** - RESTful 规范
5. **风险评估** - 识别并缓解风险

## 输出

- 技术设计 → `main/docs/tech_designs/[功能名].md`
- API 规范 → `main/docs/api/[功能名].yaml`
