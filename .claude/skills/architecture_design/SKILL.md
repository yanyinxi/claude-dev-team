# 架构设计技能

## 何时使用

当需要以下操作时使用此技能：
- 设计系统架构
- 选择技术栈
- 评估技术风险
- 制定技术决策

## 工作流程

### 第一步：分析 PRD
- 理解产品需求
- 识别技术挑战
- 评估可行性

### 第二步：架构设计
- 选择架构模式（单体、微服务、Serverless）
- 定义系统组件和模块
- 设计组件间通信方式

### 第三步：技术选型
- 前端技术栈（React、Vue、Next.js）
- 后端技术栈（Node.js、Python、Go、Java）
- 数据库选择（PostgreSQL、MySQL、MongoDB、Redis）
- 基础设施（部署、监控、日志）

### 第四步：风险评估
- 识别技术风险
- 评估影响和概率
- 制定缓解措施

## 输出格式

```markdown
# 技术设计文档：[功能名称]

## 1. 架构概述
- 架构模式
- 系统组件
- 组件关系

## 2. 技术栈
- 前端：[技术栈]
- 后端：[技术栈]
- 数据库：[数据库]
- 基础设施：[基础设施]

## 3. API 设计
- 端点列表
- 请求/响应格式
- 认证方式

## 4. 数据库设计
- 表结构
- 关系
- 索引策略

## 5. 风险评估
- 风险 1：[描述] → [缓解措施]
```

## 最佳实践

1. **简单优先**：避免过度设计
2. **可扩展**：考虑未来扩展需求
3. **安全第一**：在架构中内置安全考虑
