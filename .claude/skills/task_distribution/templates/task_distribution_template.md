## 复杂度评估标准

| 复杂度 | 评分 | 描述 | 示例 |
|--------|------|------|------|
| 简单 | 1-3 | 基础功能，1-2 小时完成 | 简单页面、基础 CRUD |
| 中等 | 4-6 | 需要设计，半天到1天 | 复杂表单、业务逻辑 |
| 复杂 | 7-8 | 多方协调，1-2 天 | 权限系统、支付集成 |
| 非常复杂 | 9-10 | 系统级改动，超过 2 天 | 架构重构、性能优化 |

## 风险和缓解措施

### 风险1: 权限验证中间件复杂度高
- **影响**: 可能延期1天
- **缓解**: 提前研究现有实现，准备备用方案

### 风险2: 前后端接口对接可能有问题
- **影响**: 可能需要额外调试时间
- **缓解**: 先定义清晰的 API 契约，使用 Mock 数据并行开发
